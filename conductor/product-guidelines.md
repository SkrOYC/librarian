# Product Guidelines: Librarian CLI - Technology Research Agent

## Prose Style and Tone
- Technical and precise, focusing on accuracy and completeness in all communications
- Use formal technical language appropriate for software engineering contexts
- Prioritize factual accuracy over explanatory examples when there's a trade-off
- Provide comprehensive information that covers all relevant aspects of a query
- Maintain consistency in terminology and technical descriptions

## Brand Messaging
- Position as reliable and trustworthy, providing dependable access to accurate code information
- Emphasize the reduction of hallucinations through grounding in actual source code
- Highlight the tool's role as a dependable reference for AI agents
- Focus on trustworthiness as the primary value proposition
- Communicate dependability in all documentation and responses

## User Experience for AI Agents
- Fast and efficient with minimal overhead in communication
- Optimize for quick response times and streamlined interactions
- Reduce latency in information retrieval processes
- Provide efficient data formats that are easy for AI agents to parse
- Minimize the number of steps required to get accurate information

## Quality Standards
- Accuracy as the primary quality standard, ensuring all information provided is grounded in actual source code
- Verify all responses against actual repository content before delivery
- Prioritize correctness over speed when there's a trade-off
- Implement validation mechanisms to ensure information fidelity
- Maintain a zero-tolerance approach to hallucinations in responses

## Development Principles
- All features should support the core mission of providing accurate, reliable access to codebase information
- Design APIs and interfaces with automation and AI agent usage in mind
- Maintain backward compatibility to ensure stable AI agent workflows
- Implement comprehensive error handling and graceful degradation
- Focus on performance optimization without compromising accuracy
